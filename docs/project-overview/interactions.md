## 2. addf-dev の全フェーズフロー図

```
/addf-dev 起動
│
▼
┌─────────────────────────────┐
│ Step 1: コンテキスト読み込み    │
│ Feedback.md → TODO.md →     │
│ Progress.md                  │
└──────────┬──────────────────┘
           │
           ▼
┌─────────────────────────────┐
│ Step 2: タスク選択            │
│ 優先度: ブロッカー > インフラ   │
│ > 若番                       │
└──────────┬──────────────────┘
           │
           ▼
┌─────────────────────────────┐
│ Step 3: knowhow 参照         │
│ addf-knowhow-agent          │
│ └─ Plan に関連する知見を注入   │
└──────────┬──────────────────┘
           │
           ▼
┌─────────────────────────────┐
│ Step 4: 実装                 │
│ Progress.md にチェックリスト   │
│ サブタスク単位で実装           │
│ 並列可能なら git worktree     │
└──────────┬──────────────────┘
           │
           ▼
┌─────────────────────────────────────────────┐
│ Step 5: 品質検証                              │
│                                              │
│ Stage 1: ビルド検証 ◀──── 失敗時ループ        │
│ ├─ bash .claude/tests/run-all.sh             │
│ └─ プロジェクト固有 build/lint/test            │
│         │                                     │
│         ▼ (通過)                               │
│ Stage 2: 品質検証チーム（並列）                  │
│ ├─ addf-code-review-agent ──┐                │
│ ├─ addf-security-review ────┤─ 集約          │
│ └─ addf-contribution ───────┘                │
│         │                                     │
│ Critical/High → 修正 → Stage 1 再実行         │
└──────────┬──────────────────────────────────┘
           │
           ▼
┌─────────────────────────────┐
│ Step 6: 完了処理              │
│ ├─ Plan 完了状況反映          │
│ ├─ /addf-knowhow 記録        │
│ ├─ Feedback.md 更新          │
│ ├─ Progress.md アーカイブ     │
│ └─ コミット                   │
└──────────┬──────────────────┘
           │
           ▼
  /loop 時: 次のタスクへ
```

## 3. 品質ゲートフロー図

```
実装完了
│
▼
┌───────────────────────────────────┐
│ Stage 1: ゲートキーパー             │
│                                    │
│ bash .claude/tests/run-all.sh     │
│ + プロジェクト固有 build/lint/test   │
│                                    │
│ ┌──────┐    ┌──────┐             │
│ │ PASS │    │ FAIL │──▶ 差し戻し  │
│ └──┬───┘    └──────┘    (修正後   │
│    │                     再実行)   │
└────┼──────────────────────────────┘
     │
     ▼
┌───────────────────────────────────────────┐
│ Stage 2: 品質検証チーム（並列起動）           │
│                                            │
│ ┌─────────────────┐  ┌────────────────┐  │
│ │ code-review     │  │ security-review│  │
│ │ (Sonnet)        │  │ (Sonnet)       │  │
│ │                 │  │                │  │
│ │ 品質/可読性/    │  │ 脆弱性検出     │  │
│ │ ベストプラクティス│  │ 修正案提示     │  │
│ └────────┬────────┘  └───────┬────────┘  │
│          │                    │            │
│ ┌────────┴────────┐  ┌───────┴────────┐  │
│ │ contribution    │  │ ui-test-agent  │  │
│ │ (Sonnet)        │  │ (Sonnet)       │  │
│ │                 │  │ [オプション]    │  │
│ │ upstream 貢献   │  │ GUI 視覚検証   │  │
│ │ 候補検出        │  │                │  │
│ └────────┬────────┘  └───────┬────────┘  │
└──────────┼────────────────────┼────────────┘
           │                    │
           ▼                    ▼
┌───────────────────────────────────┐
│ フィードバック集約                   │
│                                    │
│ Critical/High → 即修正 → Stage 1   │
│ Medium → 修正 or 独立計画           │
│ Low/Info → 計画に記録               │
└───────────────────────────────────┘
```
